🔍 RSI Discovery Engine — Architecture

Fase 1 (Priority C): Mark menemukan repo relevant tanpa API key

Flow: GitHub → Filter → Analyze → Suggest
Step 1: Seed Sources
Lo definisiin seed list: GitHub stars lo, dependencies existing project, kategori (web scraping, document parsing, system monitoring). Mark gak blind crawl — mulai dari trusted sources.
↓ Existing browser-agent.js → navigate + readDOM
Step 2: Browser-based GitHub Crawler
Mark pake browser bawaan (Chromium di electron) buat buka GitHub.
✅ Gak perlu GitHub API key — rate limit browser jauh lebih longgar.
✅ Bisa scrape trending, search results, user profile, README full content.
✅ Reuse browser-navigate, browser-read, browser-click yang udah ada.
↓ Parse HTML → extract metadata (stars, topics, last commit)
Step 3: Tier-1 Filter (Cepat, tanpa LLM)
Filter by: stars > threshold, last commit < 6 bulan, license, language.
⚡ Tujuannya: buang 90% noise tanpa sentuh LLM.
↓ README masuk ke proses embedding + scoring
Step 4: Tier-2 Semantic Match
Baca README → embedding pake Transformers.js (udah ada di Mark).
Compare dengan "Capability Profile Vector" Mark — representasi dari:
• Semua plugin yang terinstall
• Tool description dari native-tools.js
• Usage history (sering ngerjain task apa)
↓ Hasil ranking → saran ke user
Step 5: Suggestion + Auto-Attempt (Hybrid)
Mark nyajiin report: "Gue nemu 3 repo yang relevan buat nambahin X".
• Low-risk (tools, plugins) → Mark cobain clone + adaptasi langsung
• High-risk (core architecture changes) → minta approval lo dulu
